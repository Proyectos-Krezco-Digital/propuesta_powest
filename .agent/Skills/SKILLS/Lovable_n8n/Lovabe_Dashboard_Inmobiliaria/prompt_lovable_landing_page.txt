Quiero que crees una landing page moderna y responsive para una inmobiliaria llamada "InmoNova", enfocada en captación de leads de calidad.

Objetivo principal:
- Mostrar una página clara y persuasiva para que usuarios interesados en comprar, alquilar o invertir en inmuebles rellenen un formulario.
- El formulario debe enviar los datos a un workflow de n8n a través de la integración MCP.
- El workflow de n8n que debes usar se llama exactamente: "Procesar Leads".

⚙️ Integración con n8n (MCP):
- Utiliza la herramienta MCP de n8n integrada en Lovable.
- Al enviar el formulario, haz una llamada al workflow "Procesar Leads".
- Envía los datos del formulario como un objeto JSON con estas claves exactas:

  {
    "Nombre": string,
    "Email": string,
    "Telefono": string,
    "QueQuieresHacer": string,
    "TipoInmueble": string,
    "ZonaBarrio": string,
    "Presupuesto": string,
    "Plazo": string,
    "ComoNosHasConocido": string,
    "Mensaje": string
  }

- No hace falta que definas la URL; debes usar la integración MCP de n8n y el nombre del workflow "Procesar Leads".
- Espera como respuesta un objeto JSON con el lead enriquecido (campos originales + campos IA). No necesitas mostrar todos los campos de respuesta, pero sí usar la respuesta para mostrar un mensaje de confirmación personalizado.

🧾 Campos del formulario (frontend):

Crea un formulario con estos campos y nombres de variable exactamente iguales a las claves anteriores:

1. Nombre (input texto)
   - Label: "Nombre completo"
   - Placeholder: "Ej. Ana López"
   - Requerido.

2. Email (input email)
   - Label: "Email"
   - Placeholder: "ejemplo@correo.com"
   - Requerido.
   - Validación básica de formato de email.

3. Telefono (input texto / tel)
   - Label: "Teléfono"
   - Placeholder: "Ej. +34 600 123 456"
   - Requerido.

4. QueQuieresHacer (select)
   - Label: "¿Qué quieres hacer?"
   - Opciones (values de los option deben ser exactamente estos textos):
     - "Comprar vivienda"
     - "Alquilar vivienda"
     - "Invertir en inmuebles"
     - "Otro"
   - Requerido.

5. TipoInmueble (select)
   - Label: "Tipo de inmueble"
   - Opciones:
     - "Piso"
     - "Casa / chalet"
     - "Apartamento / estudio"
     - "Estudio"
     - "Local comercial"
     - "Oficina"
     - "Otro"
   - Requerido.

6. ZonaBarrio (input texto)
   - Label: "Zona o barrio de interés"
   - Placeholder: "Ej. Centro Madrid, Malasaña, Sevilla Triana..."
   - Requerido.

7. Presupuesto (input texto)
   - Label: "Presupuesto aproximado"
   - Placeholder: "Ej. 350.000 €, hasta 500.000 €, entre 400.000 y 450.000 €"
   - Requerido.

8. Plazo (select)
   - Label: "Plazo en el que te gustaría cerrar la operación"
   - Opciones:
     - "Lo antes posible (0–3 meses)"
     - "En 3–6 meses"
     - "Más de 6 meses"
     - "Solo estoy mirando opciones"
   - Requerido.

9. ComoNosHasConocido (select)
   - Label: "¿Cómo nos has conocido?"
   - Opciones:
     - "Google"
     - "Redes sociales"
     - "Portal inmobiliario"
     - "Recomendación"
     - "Otro"
   - Opcional, pero si se rellena se envía igualmente en el JSON.

10. Mensaje (textarea)
    - Label: "Cuéntanos qué estás buscando"
    - Placeholder: "Ej. Busco piso de 3 habitaciones en Madrid centro, con garaje, presupuesto máximo 450.000 €, para mudarme antes de verano."
    - Requerido.
    - Este campo es especialmente importante para el workflow de IA de n8n.

🔁 Comportamiento del formulario:

- Al hacer submit:
  - Validar campos requeridos en el frontend.
  - Mostrar estado de "enviando" (botón deshabilitado + spinner o mensaje "Procesando tu solicitud...").
  - Llamar al workflow de n8n "Procesar Leads" usando la integración MCP, enviando el JSON con los campos especificados.

- Si la respuesta es correcta:
  - Mostrar una pantalla o bloque de éxito en la propia landing.
  - Mensaje sugerido:
    - Título: "¡Hemos recibido tu solicitud!"
    - Texto: "Uno de nuestros agentes revisará tu caso y se pondrá en contacto contigo muy pronto."
  - Si en la respuesta viene un campo tipo "ResumenIA", puedes mostrarlo en pequeño como:
    - "Resumen de tu solicitud: [ResumenIA]"
  - Limpiar el formulario o deshabilitarlo para evitar reenvíos accidentales.

- Si hay error en la llamada:
  - Mostrar un mensaje de error amable:
    - "Ha ocurrido un problema al enviar tu solicitud. Inténtalo de nuevo en unos minutos."
  - No borres los datos introducidos para que el usuario no tenga que reescribirlo todo.
  - Permitir reintentar el envío.

🎨 Diseño de la landing:

Estilo general:
- Diseño limpio, moderno, profesional, estilo SaaS inmobiliario.
- Paleta de colores:
  - Color principal: un azul moderno (#2563eb o similar) para botones y acentos.
  - Fondo claro (blanco o gris muy suave).
  - Tipografía sans serif elegante (por ejemplo, similar a Inter / SF Pro).
- Debe ser totalmente responsive (móvil primero).

Estructura recomendada:

1. Hero principal (parte superior):
   - Título grande: "Encuentra tu próxima vivienda con asesoramiento experto."
   - Subtítulo: "Déjanos tus datos y un agente de InmoNova analizará tu caso y te propondrá las mejores opciones según tu perfil."
   - A la derecha (en desktop) o debajo (en móvil), el formulario en una tarjeta con fondo blanco y sombra suave.

2. Sección de beneficios:
   - Tres o cuatro columnas con iconos y textos cortos:
     - "Asesoramiento personalizado con IA"
     - "Selección de inmuebles filtrada por tus necesidades"
     - "Respuesta rápida de un agente especializado"
   - No hace falta funcionalidad extra, solo contenido estático.

3. Sección de confianza:
   - Podrías añadir logotipos de portales donde aparece la inmobiliaria (simulados) o texto tipo:
     - "Más de 500 clientes han confiado en InmoNova para encontrar su vivienda."

4. Pie de página:
   - Datos básicos: nombre de la inmobiliaria, aviso de privacidad, enlace a política de protección de datos (puede ser un enlace placeholder "#").

💡 Detalles de UX:

- Marca claramente los campos obligatorios con un asterisco.
- Muestra mensajes de error cerca de cada campo que falle la validación.
- Usa texto de ayuda debajo de algunos campos (por ejemplo en Presupuesto y Mensaje) para guiar al usuario.
- El botón de envío debe ser claramente visible:
  - Texto sugerido: "Enviar mi solicitud" o "Quiero que me contacten".
- En móvil, el formulario debe ocupar prácticamente todo el ancho, con buen margen y padding.

En resumen:
- Construye una landing de captación de leads para la inmobiliaria "InmoNova".
- Incluye el formulario con los campos descritos.
- Conecta el submit del formulario al workflow de n8n "Procesar Leads" usando la integración MCP, enviando los datos en el JSON con las claves indicadas.
- Gestiona estados de loading, éxito y error de forma clara para el usuario.
